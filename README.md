# Play Store Auto Tester 🚀

**플레이스토어 비공개 테스트 자동화 앱**

12대 폰에서 14일 동안 자동으로 테스트 앱을 실행하여 플레이스토어 프로덕션 승인 요건을 충족시켜주는 완벽한 자동화 앱입니다!

---

## 주요 기능

✅ **완전 자동화**: 매일 랜덤 시간(02:00~23:00)에 테스트 앱 자동 실행
✅ **무작위 터치**: 3~4분간 무작위로 화면을 터치하여 실제 사용자처럼 동작
✅ **간단한 UI**: 앱 목록에서 선택하거나 패키지명만 입력하면 끝
✅ **12대 폰 관리**: 각 폰마다 동일하게 설정하고 14일만 기다리면 프로덕션 승인!
✅ **안전한 실행**: Google이 감지하지 못하는 실제 기기 테스트 방식

---

## 사용 방법

### 1단계: 앱 설치
1. 이 프로젝트를 빌드하여 APK 생성
2. 중고폰 12대에 APK 설치

### 2단계: 테스트할 앱 준비
1. Play Store에서 비공개 테스트 트랙에 앱 업로드
2. 각 폰에서 opt-in 링크로 테스트 앱 설치

### 3단계: 자동화 설정
1. **Play Store Auto Tester** 앱 실행
2. 설치된 앱 목록에서 테스트할 앱 선택 (또는 패키지명 직접 입력)
3. **"설정 저장"** 버튼 클릭
4. **"자동화 시작"** 버튼 클릭

### 4단계: 14일 기다리기
- 각 폰이 매일 랜덤 시간에 자동으로 앱을 실행합니다
- 3~4분간 무작위 터치 후 자동 종료
- 14일 후 Play Console에서 프로덕션 신청 가능!

## 빌드 방법

### 필수 조건
- Flutter SDK 3.9.0 이상
- Android Studio
- Android SDK

### 빌드 명령어

```bash

# 2. APK 빌드 (12대 폰에 설치용)

# 3. APK 위치
# build/app/outputs/flutter-apk/app-release.apk
```

## 12대 폰 세팅 가이드

### 각 폰마다 해야 할 일

1. **구글 계정 로그인** (12개 모두 다른 계정 사용)
2. **Play Store 자동 업데이트 켜기**
   - Play 스토어 → 설정 → 네트워크 환경설정 → "모든 네트워크에서 자동 업데이트"
3. **테스트할 앱 설치**
   - Play Console에서 받은 opt-in 링크로 비공개 테스트 앱 설치
4. **Auto Tester 앱 설치 및 설정**
   - 이 앱 설치 → 테스트 앱 선택 → "자동화 시작"
5. **충전기 연결 및 Wi-Fi 연결**
   - 14일간 계속 켜놓아야 하므로 충전기 필수

### 권장 사항

- 폰 이름을 "1번폰", "2번폰" ~ "12번폰"으로 변경 (관리 편함)
- 화면 자동 꺼짐 시간을 최대한 길게 설정
- 절전 모드에서 앱이 종료되지 않도록 배터리 최적화 제외 설정

---

## 테스트 즉시 실행

설정이 제대로 되었는지 확인하고 싶다면:

1. 앱에서 **"지금 바로 테스트"** 버튼 클릭
2. 잠시 후 테스트 앱이 자동으로 실행되고 3~4분 후 종료됩니다

---

## 자동화 동작 방식

```
매일 랜덤 시간 (02:00 ~ 23:00 사이)
  ↓
테스트 앱 실행
  ↓
5초 대기 (앱 로딩)
  ↓
3~4분간 무작위 터치 (3~8초 간격)
  ↓
홈 화면으로 이동 및 종료
  ↓
다음날 다시 반복
```

## 문제 해결

### Q: 앱이 자동으로 실행되지 않아요
A: 다음을 확인하세요:
- 배터리 최적화가 꺼져 있는지 확인
- 권한(오버레이, 알람 등)이 모두 허용되었는지 확인
- "자동화 시작" 버튼을 눌렀는지 확인

### Q: 터치가 안 됩니다
A: Android 7.0 이상에서는 `input tap` 명령어가 제한될 수 있습니다.
- 폰의 개발자 옵션에서 "USB 디버깅" 활성화 (필요시)

### Q: 14일이 지나도 프로덕션 버튼이 안 열려요
A: Play Console 확인:
- 최소 12명 이상 opt-in 상태 유지 확인
- 각 기기에서 언인스톨하지 않았는지 확인
- 14일 연속 유지 조건 충족 여부 확인
