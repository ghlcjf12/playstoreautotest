# 빠른 시작 가이드 ⚡

## 5분 만에 시작하기

### 1단계: APK 빌드 (2분)

```bash
# 프로젝트 폴더에서 실행
flutter build apk --release
```

빌드가 완료되면:
- **APK 위치**: `build/app/outputs/flutter-apk/app-release.apk`
- 이 파일을 12대 폰에 설치하세요

---

### 2단계: 각 폰 세팅 (폰당 1분)

#### 2-1. APK 설치
- `app-release.apk` 파일을 폰으로 전송 (USB, 카카오톡, 이메일 등)
- 설치 (출처를 알 수 없는 앱 허용 필요)

#### 2-2. Play Store에서 테스트 앱 설치
1. Play Console에서 받은 opt-in 링크로 접속
2. 비공개 테스트 참여
3. 테스트할 앱 설치

#### 2-3. Auto Tester 앱 실행
1. **Play Store Auto Tester** 앱 열기
2. 권한 허용 (알람, 오버레이 등)
3. 설치된 앱 목록에서 테스트할 앱 **선택** 또는 패키지명 직접 입력
4. **"설정 저장"** 버튼 클릭
5. **"자동화 시작"** 버튼 클릭

#### 2-4. 테스트 (선택사항)
- **"지금 바로 테스트"** 버튼으로 제대로 동작하는지 확인
- 잠시 후 테스트 앱이 자동 실행되고 3~4분 후 종료되면 성공!

---

### 3단계: 14일 기다리기

- 각 폰을 충전기에 연결하고 Wi-Fi 켜두기
- 매일 랜덤 시간(02:00~23:00)에 자동으로 테스트 앱 실행
- 14일 후 Play Console 확인 → 프로덕션 신청 가능!

---

## 패키지명 찾는 법

모르겠으면 아래 방법으로 확인:

1. **Play Console에서**:
   - Play Console → 내 앱 선택 → 왼쪽 메뉴 설정
   - "애플리케이션 ID" 확인 (예: `com.example.myapp`)

2. **폰에서 직접**:
   - Play Store에서 앱 상세페이지 열기
   - URL 확인: `https://play.google.com/store/apps/details?id=여기가_패키지명`

3. **Android Studio에서**:
   - `AndroidManifest.xml` 열기
   - 맨 위 `<manifest package="여기가_패키지명">` 확인

## 확인 방법

### Play Console에서 확인
1. Play Console 로그인
2. 내 앱 선택
3. 좌측 메뉴 → **테스트 → 비공개 테스트**
4. "테스터" 탭에서 참여자 수 확인
5. 14일 연속 12명 이상 유지되면 **프로덕션 탭에서 "프로덕션으로 승격" 버튼 활성화**

---

## 문제 발생 시

### 앱이 자동으로 실행 안 됨
```
설정 → 앱 → Play Store Auto Tester → 배터리
→ "배터리 사용량 최적화" 끄기
```

### 터치가 안 됨
```
설정 → 개발자 옵션 → USB 디버깅 활성화
(개발자 옵션 없으면: 설정 → 휴대전화 정보 → 빌드 번호 7번 연타)
```

### 권한 오류
```
설정 → 앱 → Play Store Auto Tester → 권한
→ 모든 권한 허용
```
